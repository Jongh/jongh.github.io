## 3. 섹션별 콘텐츠 상세

### § 1 — 히어로 (`#hero`)

**목적:** 방문자의 첫인상. 이 사람이 누구인지를 3초 안에 파악하게 한다.

**구성 요소:**
- 이름 (h1 상단 소형 레이블 또는 h1 내 포함)
- 직함 또는 역할 (h1 — 구체적 포지셔닝)
- 한 줄 철학 (서브헤드 — AI를 다루는 방식 중심)
- CTA 버튼 2개: 「주요 업무 보기 →」(`#key-projects`) / 「연락하기」(`#contact`)

**내러티브 전략:**  
v1은 "방식을 설계합니다"는 추상적 선언이었다. v2는 이름+직함으로 실체를 먼저 드러내고, 방식 철학을 서브헤드로 보조한다.

**레이아웃:**  
- 세로 중앙 정렬, 최소 높이 `100svh`
- 이름/직함 → 철학 → CTA 수직 스택
- 배경: `--bg` (아이보리), 섹션 하단 `--border` 구분선

---

### § 2 — 자기소개 (`#about`)

**목적:** 방문자가 "이 사람과 일하면 어떨까"를 판단할 수 있는 배경·철학·차별점 제공.

**구성 요소:**
- 섹션 제목: "자기소개" (또는 "About")
- 2열 레이아웃 (데스크탑 기준):
  - **왼쪽 (1/3):** 이미지 또는 이니셜 아바타 블록 (옵션)
  - **오른쪽 (2/3):** 텍스트 블록

**텍스트 블록 구조:**
1. **포지셔닝 한 줄** (강조 — 굵게): 나는 어떤 사람인가
2. **배경 단락** (2~3문장): 어떤 경로로 지금에 이르렀는가
3. **차별점** (1~2문장 또는 키워드 3개): 다른 사람과 무엇이 다른가

**이미지 처리 원칙:**
- 실제 이미지가 없을 경우: 이니셜 텍스트 블록 (CSS only, 이미지 파일 없음)
- 이미지 제공 시: `<img>` 태그, `aspect-ratio: 1/1`, 원형 또는 둥근 모서리

**레이아웃:**  
- 배경: `--surface` (연한 베이지)
- 모바일: 텍스트 단일 열

---

### § 3 — 업무 이력 (`#work-history`)

**목적:** 방문자가 커리어 흐름과 업무 맥락을 빠르게 파악.

**구성 요소:**
- 섹션 제목: "업무 이력"
- 타임라인 카드 목록 (수직 스택, 최신 순)

**카드 1개당 구조:**
```
[회사/조직명]                        [재직 기간]
[직무]
→ 핵심 기여 1줄
```

**레이아웃:**
- 왼쪽 세로 타임라인 선 (`--border`) + 카드 오른쪽 배치
- 각 카드: 배경 `--bg`, 테두리 `--border`, 패딩 `1.5rem`
- 기간: 소형 레이블 (`--text-2`, 우측 정렬)
- 모바일: 타임라인 선 숨김, 카드 전체 너비

**배경:** `--bg` (아이보리, 밝은 섹션)

---

### § 4 — 주요 업무 (`#key-projects`)

**목적:** AI 협업 역량이 실제 업무에 어떻게 적용되었는지 구체적으로 보여준다.

**구성 요소:**
- 섹션 제목: "주요 업무"
- 섹션 서브텍스트: AI 방법론 관점에서의 사례 소개임을 안내
- 카드 그리드 (3~5개)

**카드 1개당 구조:**
```
[역량 태그들] ← 소형 배지, 상단 좌측
[프로젝트 제목] ← h3
[배경] 어떤 문제 상황이었는가 — 1~2문장
[방법] AI와 어떻게 협업했는가 — 2~3문장 (방법론 관점 필수)
[결과] 측정 가능한 임팩트 — 강조 블록 또는 수치 하이라이트
```

**레이아웃:**
- 그리드: `repeat(auto-fill, minmax(320px, 1fr))`
- 카드 배경: `--bg`, 테두리: `--border`, hover 시 `--accent` 테두리
- 역량 태그: `--accent-a` 배경, `--accent` 텍스트
- 결과 블록: `border-left: 3px solid --accent` 강조

**배경:** `--surface` (연한 베이지)

**보편성 원칙 적용:**
- 카드 내 도구명이 등장할 경우 → 방법론 관점 서술 필수
- 역량 태그는 반드시 7가지 역량 목록 내에서만 선택

---

### § 5 — AI 활용 특장점 (`#ai-strengths`)

**목적:** 방문자가 "이 사람과 AI 협업을 하면 무엇이 다른가"를 카드 단위로 파악.

**v1 `#competencies`와의 차이:**

| v1 (Competencies) | v2 (AI Strengths) |
|-------------------|-------------------|
| "보편적 AI 협업 역량" — 방법론 교육 느낌 | "AI 활용 특장점" — 나의 역량 소개 |
| 정의(What) + 효과(Why) | 정의(What) + 실증(Evidence) — 실제 적용 사례 1줄 추가 |
| 배경·철학 설명 중심 | 이 사람에게 기대할 수 있는 결과 중심 |

**카드 1개당 구조:**
```
[번호 배지]
[역량명] ← h3
[정의] — 이 역량이 무엇인가 (1~2문장)
[실증] — 실제로 어떻게 적용했는가 (1문장, Key Projects 연결)
[효과] — 결과적으로 무엇이 달라지는가 (1문장, 강조색)
```

**레이아웃:**
- 그리드: `repeat(auto-fill, minmax(300px, 1fr))` — v1과 동일
- 배경: `--surface`
- 번호 배지: `--accent-a` 배경, `--accent` 텍스트

---

### § 6 — AI 협업 방식 (`#workflow`)

**목적:** 역량이 독립적이 아닌 실제 작업 흐름 속에서 어떻게 맞물리는지 시각화.

**v1 대비 변경:**
- 내용은 유지, 시각 단순화 (베이지 테마에서 명도 차이로 단계 구분)
- 파이프라인 4단계: 설계 → 구현 → 검토 → 배포 (동일)
- 각 단계의 역량 번호 태그 유지

**레이아웃:**
- 배경: `--bg`
- 단계 박스 배경: `--surface`, 테두리: `--border`
- 화살표: `--text-2` 색

---

### § 7 — 연락처 / 푸터 (`#contact`)

**목적:** 방문자가 연락하거나 추가 정보를 탐색할 수 있는 출구 제공.

**구성 요소 (기존 유지):**
- 연락 안내 문구
- GitHub · Instagram · Email 링크
- 저작권: `© 2025 JongH`

## 4. 내러티브 흐름 (섹션 간 연결)

```
히어로 (누구인가)
    ↓
자기소개 (어떻게 이 사람이 되었는가)
    ↓
업무 이력 (어디서 무엇을 해왔는가)
    ↓
주요 업무 (실제로 무엇을 만들었는가 — AI 방법론 적용 사례)
    ↓
AI 특장점 (그 경험에서 검증된 나의 역량은 무엇인가)
    ↓
협업 방식 (함께 일하면 어떤 방식으로 진행되는가)
    ↓
연락처 (함께 일하고 싶다면)
```

방문자가 "이 사람 → 이 사람의 이력 → 이 사람이 한 일 → 이 사람이 잘하는 것 → 함께 일하는 방식 → 연락"으로 자연스럽게 이동하도록 설계.

## 6. 구현 범위 제약

- 빌드 도구 없음: `index.html` + `style.css` + `main.js` (3파일)
