# tide 개발 사이클 규약

이 문서는 tide 워크플로우(`/tide:*`)가 이 저장소에서 따르는 규약을 정의한다.
마일스톤·보고서의 **형식**은 여기서 정의하지 않는다 — tide 플러그인 각 스킬에 동봉된
`template.md`가 단일 원본(SSOT)이다.

---

## 단계별 금지 행위

| 단계 | 스킬 | git 작업 | 비고 |
|------|------|----------|------|
| 킥오프 | `/tide:kickoff` | ❌ 금지 | 골격 생성만 |
| 마일스톤 | `/tide:milestone` | ❌ 금지 | 다음 마일스톤 문서 생성 |
| 구현 | `/tide:impl` | ❌ **금지** | 구현 + 테스트 + 완료보고서. 커밋·리뷰 없음 |
| 리뷰 | `/tide:review` | ❌ **금지** | 비판적 리뷰 + 보고서 + 릴리즈 판정. 커밋·태그·푸시 없음 |
| 릴리즈 | `/tide:release` | ✅ 허용 | 유일하게 git 작업(commit·tag·push)을 수행하는 단계 |

- **impl·review 단계에서는 어떤 git 작업도 하지 않는다.** 모든 커밋/태그/푸시는 release로 모은다.
- git 작업 차단 가드(tide-guard)는 tide 플러그인이 hook으로 직접 제공하므로 프로젝트별 설치 불필요.

---

## 상태 파일 규약

| 파일 | 커밋 | 역할 |
|------|------|------|
| `.tide/phase` | ❌ `.gitignore` 처리 | 현재 사이클 단계를 기록하는 **로컬** 상태 파일 |
| `.tide/deps` | ✅ 커밋 가능 | 레포 간 의존을 한 줄에 형제 레포명 하나씩 선언하는 **선택·옵트인** 매니페스트 |

- `.tide/deps`는 선언 시 fleet이 위상정렬해 권장 처리 순서를 제시한다. 없으면 의존 0(기본 동작).
  kickoff가 이 파일을 자동 생성하지는 않는다.

---

## 태스크 ID · 의존성 표기

- 태스크 ID: 마일스톤 문서 내에서 안정적인 짧은 식별자를 부여한다 (예: `T1`, `T2` …).
- 의존성: 태스크가 다른 태스크에 의존하면 `(deps: T1, T2)` 형태로 표기한다.
- deps가 없는 태스크는 병렬 처리 후보, deps가 있으면 선행 태스크 완료 후 순차 처리한다.

---

## 버전 파일 위치

- **이 저장소에는 버전 파일이 없다.** (Cargo.toml / package.json / pyproject.toml /
  `.claude-plugin/plugin.json` 등 부재 — 빌드 도구·패키지 매니저 비종속 순수 정적 사이트)
- 릴리즈 버전 관리가 필요해지면 이 절을 갱신하고 버전 파일 위치를 명시한다.
- 현재 버전 추적은 `CHANGELOG.md` 및 git 태그로 수행한다.

---

## 마일스톤 / 보고서 형식

- 마일스톤 문서: `docs/milestones/` 에 보관.
- 보고서(완료·리뷰): `docs/reports/` 에 보관.
- **형식의 단일 원본은 tide 플러그인 각 스킬에 동봉된 `template.md`다.** 이 문서에서 형식을
  복제·재정의하지 않는다.

---

## 멀티 레포 / 대상 레포

- 모든 산출물(골격·`docs/project-context.md`·`.gitignore`·`CHANGELOG.md` 등)은 **대상 레포
  루트**에 만든다.
- 기본은 세션 레포(현행 단일 레포 동작). 상위 폴더 단일 세션에서 특정 자식 레포를 지시받으면
  그 자식 레포 루트가 대상이 된다.
- 각 레포의 `.tide/` 상태는 자기 레포 루트에 격리된다 — 레포 간 상태를 공유하지 않는다.

### 멀티 레포 오케스트레이션 (발견)

- **세는 대상**: 현재 세션 위치(cwd)의 **직속 1단계 하위 디렉터리** 중 **자식 tide 레포**의 개수.
- **자식 tide 레포 판별**: 하위 디렉터리가 자체 git 레포이면서 tide 골격(예: `docs/conventions.md`
  또는 `docs/milestones/`)을 가진 경우.
- **제외**: 이름이 `.`으로 시작하는 숨김 디렉터리는 세지 않는다.
- 자식 tide 레포가 2개 이상이면 `/tide:fleet`(교차 개요)·`/tide:fleet-cycle`·`/tide:fleet-verify`로
  교차 오케스트레이션이 가능하다. advisory일 뿐 자동 실행하지 않는다.
