## § 2. 관점 도입부 (`#perspective`)

**섹션 레이블 (소문자 강조)**
```
왜 언어가 아니라 방식인가
```

**본문**
```
"어떤 언어 쓰세요?" 라는 질문은 자연스럽습니다.
하지만 AI와 함께 일하는 시대에 진짜 질문은 달라집니다.

언어와 도구는 매년 바뀝니다.
모델도, 플랫폼도, 생태계도 교체됩니다.
그 속에서 지속적으로 가치를 만들어내는 것은
"무엇을 쓰는가"가 아니라 "어떻게 다루는가"입니다.

이 페이지는 도구 목록 대신, 방식을 소개합니다.
```

## § 3. 역량 7가지 (`#competencies`)

**섹션 제목**
```
보편적 AI 협업 역량
```

**섹션 서브텍스트**
```
어떤 AI를 쓰든, 어떤 도메인에서 일하든 적용되는 협업의 원칙들입니다.
```

---

### 카드 1 — 컨텍스트 엔지니어링

```
제목:   컨텍스트 엔지니어링
정의:   AI가 올바른 판단을 내리도록
        배경·제약·목표를 구조화된 언어로 설계합니다.
효과:   반복 수정 없이 첫 시도에서 원하는 결과를 얻는 효율을 확보합니다.
```

### 카드 2 — 작업 분해

```
제목:   작업 분해
정의:   복잡한 목표를 AI가 처리 가능한 최소 단위로 나누고
        실행 순서를 설계합니다.
효과:   대형 프로젝트도 예측 가능한 단계로 진행되어
        품질 관리가 가능해집니다.
```

### 카드 3 — 역할 분업 (오케스트레이션)

```
제목:   역할 분업
정의:   여러 AI 에이전트 또는 페르소나에 역할을 배분하고
        각각의 출력을 하나의 결과물로 통합합니다.
효과:   단일 모델의 한계를 넘어
        전문화된 시각들이 수렴된 결과를 만듭니다.
```

### 카드 4 — 검증·승인 게이트

```
제목:   검증·승인 게이트
정의:   AI 출력물이 다음 단계로 넘어가기 전
        사람의 판단이 개입하는 체크포인트를 설계합니다.
효과:   오류가 누적되기 전에 조기 발견되어
        전체 재작업 비용이 최소화됩니다.
```

### 카드 5 — 검증 루프

```
제목:   검증 루프
정의:   AI가 스스로 자신의 출력을 기준에 따라
        반복적으로 검토하고 수정하는 프로세스를 설계합니다.
효과:   사람의 개입 없이도
        일정 수준 이상의 품질이 자동으로 보장됩니다.
```

### 카드 6 — 의사결정 기록

```
제목:   의사결정 기록
정의:   AI와의 협업 과정에서 내려진 판단 근거를
        구조화된 형태로 추적하고 보존합니다.
효과:   유사한 문제가 다시 발생했을 때
        동일한 사고 경로를 재현하거나 오류 원인을 빠르게 파악합니다.
```

### 카드 7 — 재사용 자산화

```
제목:   재사용 자산화
정의:   반복 사용 가능한 프롬프트·템플릿·컨텍스트 블록으로
        협업 결과를 구조화합니다.
효과:   새 프로젝트를 시작할 때 초기 비용이 줄고
        품질의 일관성이 높아집니다.
```

## § 4. 작업 흐름 시각화 (`#workflow`)

**섹션 제목**
```
AI 협업 파이프라인
```

**섹션 서브텍스트**
```
역량은 독립적으로 존재하지 않습니다.
설계부터 배포까지 각 단계에서 맞물려 작동합니다.
```

**파이프라인 레이블 4단계**

```
[설계]         →  [구현]            →  [검토]              →  [배포]
Architect          Implementer          Reviewer               Deploy

①컨텍스트         ③역할 분업           ④검증·승인 게이트      ⑥의사결정 기록
 엔지니어링        ⑦재사용 자산화       ⑤검증 루프
②작업 분해
```
